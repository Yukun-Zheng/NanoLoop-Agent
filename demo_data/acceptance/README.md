# UI acceptance fixture

The files in this directory are deterministic, project-authored engineering fixtures generated by
`scripts/generate_acceptance_fixture.py`.

| File | Purpose | SHA-256 |
| --- | --- | --- |
| `nanoloop_ui_acceptance_fixture.png` | Synthetic grayscale input with bright particles | `5827ef54f87a53bf33e6d1ab612360c70a17058c2a65d3f2457420ce773d7876` |
| `nanoloop_ui_acceptance_corrected_mask.png` | Deterministic binary mask for the immutable review-child workflow | `51adb54abfbf5229952083448c38e82383547cb994b6474902e2dda9a5082e7d` |

Both files are 2048 × 1536 px. They use no external source data and are intended for upload,
runtime, artifact, review, query, export, screenshot, and user-guide checks. They are **not
scientific data** and must not support accuracy, generalization, or materials claims.

Regenerate it from the repository root:

```bash
.venv/bin/python scripts/generate_acceptance_fixture.py
```

The files may be included in public project screenshots because they contain no private SEM pixels
or instrument metadata. The end-user walkthrough is
[the UI acceptance guide](../../docs/USER_ACCEPTANCE_GUIDE.md); its local knowledge exercise uses
[`demo_data/rag/sources/project_sample_context.md`](../rag/sources/project_sample_context.md).
